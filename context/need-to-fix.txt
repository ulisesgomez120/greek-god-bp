Notes about PWAs / web
- StorageAdapter provides web fallbacks:
  - secure.* uses sessionStorage (NOT secure) for web — this is a documented tradeoff. For production PWA, prefer using Supabase web SDK session persistence or server-set HttpOnly cookies to avoid storing long-lived tokens in localStorage/sessionStorage.
  - async.* uses localStorage on web (JSON-serialized); this provides listable keys and reasonable performance for PWA caches.
- Because you opted to start fresh (development), no legacy migration was performed. If you later want to import existing user storage, we can add a one-time migration routine.
